cap

The 'cap' method in Go is used to return the capacity of a slice, which is the maximum number of elements the slice can hold without allocating additional memory. The capacity of a slice is determined by the underlying array and can be equal to or greater than its length. This method is useful for optimizing memory usage and performance when working with slices in Go.

s = make([]int, 5, 10)  # Create a slice with length 5 and capacity 10
print(cap(s))  # Output: 10

# Using it in a function to demonstrate how capacity behaves:
def print_slice_info(slice):
    print('Length:', len(slice))  # Returns the number of elements in the slice
    print('Capacity:', cap(slice))  # Returns the maximum capacity of the slice

s1 = make([]int, 0, 5)  # Create an empty slice with capacity 5
print_slice_info(s1)  # Output: Length: 0, Capacity: 5
s = []int{1, 2, 3}
print(cap(s))  # Output: 3  # Since the length equals capacity in this case, it shows the maximum elements

# Demonstrating how appending affects the capacity:
append(s, 4)  # Appending element 4, capacity remains at 3
print(cap(s))  # Still Output: 3

append(s, 5)  # Now when we append beyond capacity it reallocates
print(cap(s))  # Output could vary depending on Goâ€™s memory allocation strategy.
s = []int{1, 2, 3, 4, 5}
print(cap(s))  # Output: 5  

# Showing effect of slicing:
new_slice = s[1:3]  # Create a new slice from the existing slice
print('New Slice Length:', len(new_slice))  # Output: 2
print('New Slice Capacity:', cap(new_slice))  # Output: 4  # Cap of the new slice is calculated based on the original.
