close

In Go, the 'close' method is used to close channels. Closing a channel indicates that no more values will be sent on it. It is important to close channels to prevent resource leaks and to notify receivers that no more data will be sent. Closing a channel also allows the range loop or select statements that read from the channel to terminate correctly. However, it's essential to note that you should only close a channel from the sending side, and if you try to close a channel that is already closed, it will result in a panic.

def producer(channel):
    for i in range(5):
        print('Producing:', i)
        channel.send(i)
    channel.close()


def consumer(channel):
    while True:
        try:
            value = channel.receive()
            print('Consuming:', value)
        except ChannelClosed:
            print('Channel closed, no more items to receive.')

def main():
    channel = Channel()

    def producer(channel):
        for i in range(10):
            channel.send(i)
        channel.close()

    def consumer(channel):
        while True:
            val = channel.receive()
            if val is None:
                break
            print('Received:', val)

    # Start both
    start(producer, channel)
    start(consumer, channel)

def close_example():
    channel = Channel()

    def generate_data(channel):
        for data in range(5):
            channel.send(data)
        channel.close()

    def handle_data(channel):
        while True:
            data = channel.receive()
            if data is None:
                print('Finished processing data.')
                break
            print('Handling data:', data)

    # Initiate data generation and handling
    start(generate_data, channel)
    start(handle_data, channel)
