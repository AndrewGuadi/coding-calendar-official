close

The 'close' method in Go is used primarily to close resources that are no longer needed, such as files, network connections, or database connections. When a resource is closed, it signals that the resource should be released and any associated cleanup logic can execute. Failing to close resources can lead to leaks and unspecified behavior in applications. It's crucial to ensure that the 'close' method is called when a resource is no longer needed, typically using a defer statement right after opening a resource.

with open('example.txt', 'r') as file:
    data = file.read() 
    print(data)
# The file is automatically closed after the block

import sqlite3

connection = sqlite3.connect('example.db')
try:
    cursor = connection.cursor()
    cursor.execute('SELECT * FROM users')
    results = cursor.fetchall()
    print(results)
finally:
    connection.close()  
# The connection to the database is closed in the finally block

import socket

server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server_socket.bind(('localhost', 8080))
server_socket.listen(1)
try:
    client_socket, addr = server_socket.accept()
    print('Connection from:', addr)
    client_socket.sendall(b'Hello, World')
finally:
    server_socket.close()
# The server_socket is closed after handling the connection
