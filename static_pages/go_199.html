cap

The 'cap' method in Go is used to return the capacity of a slice. The capacity is defined as the number of elements the slice can hold without reallocating memory. This is important for performance optimization, especially when you know the approximate size of the data the slice will hold in advance. Knowing the slice's capacity can help to reduce the overhead of memory allocation during slice growth.

s := make([]int, 0, 5)
fmt.Println(cap(s)) // Output: 5

// Here, we create a slice with a capacity of 5. The cap function returns the maximum number of elements the slice can currently hold.
fruits := []string{"apple", "banana", "cherry"}
fmt.Println(cap(fruits)) // Output: 3

// In this case, we create a slice with three elements. The cap method returns 3, which is the current capacity of the slice.
numbers := []int{1, 2, 3, 4, 5}
fmt.Println(cap(numbers)) // Output: 5

// This example shows a slice initialized with five elements. The cap method also returns 5, indicating that its capacity matches the number of elements.
