close

The 'close' method in Go is used to close a resource, such as a file or network connection, once it is no longer needed. Closing a resource is important because it releases the resources associated with it and helps prevent resource leaks. In the context of files, it ensures that all buffered data is flushed and that the file descriptor can be reused by the operating system. Always check for errors when closing resources, as the closure itself may encounter issues (like an unflushed buffer).

```python
import os

# Function to read a file and close it properly

def read_file(file_path):
    try:
        file = open(file_path, 'r')  # Open the file
        content = file.read()        # Read the content
        print(content)               # Print the content
    finally:
        file.close()                # Ensure the file is closed

# Usage
read_file('example.txt')
```
```python
import socket

# Function to create a socket, connect it, and close it properly

def connect_to_server(host, port):
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)  # Create a socket
    try:
        s.connect((host, port))  # Connect to the server
        s.sendall(b'Hello, world') # Send some data
    finally:
        s.close()                # Ensure the socket is closed

# Usage
connect_to_server('localhost', 8080)
```
```python
import sqlite3

# Function to use a database connection and close it properly

def query_database(db_path):
    conn = sqlite3.connect(db_path)   # Establish a connection to the database
    try:
        cursor = conn.cursor()          # Create a cursor object
        cursor.execute('SELECT * FROM users')  # Execute a query
        results = cursor.fetchall()      # Fetch results
        for row in results:
            print(row)                 # Print each row
    finally:
        conn.close()                  # Ensure the connection is closed

# Usage
query_database('my_database.db')
```
