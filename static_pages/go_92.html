cap

The 'cap' method in Go is used to determine the capacity of a slice, which represents the maximum number of elements that the slice can hold. The capacity of a slice can be equal to or greater than its length. This is especially useful when working with slices that may grow over time. You can use 'cap()' to plan for future memory requirements or optimizations in your program.

def slice_capacity_example():
    slice = [1, 2, 3, 4, 5]
    print('Original Slice:', slice)
    print('Capacity (in terms of list length):', len(slice))  
    # Simulating capacity logic by noting that list in Python has dynamic capacity based on usage
    slice.append(6)
    print('After appending an element:', slice)
    print('New Length:', len(slice))

def dynamic_slice_example():
    initial_slice = [10, 20, 30]
    print('Initial Slice:', initial_slice)
    print('Initial Length:', len(initial_slice))
    print('Simulated Capacity (Note: Python lists grow dynamically)')
    for i in range(4, 8):
        initial_slice.append(i * 10)
        print(f'After appending {i*10}, Length:', len(initial_slice))

def memory_management_example():
    data = [100, 200, 300]
    print('Initial Data:', data)
    print('Initial Length:', len(data))
    additional_data = [400, 500]
    data.extend(additional_data)
    print('Data after extending:', data)
    print('Final Length:', len(data))
