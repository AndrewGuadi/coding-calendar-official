close

In Go, the 'close' function is used to close a channel, preventing any further sends on that channel. It is crucial in concurrent programming as it signals to the receiving goroutines that no more data will be sent on the channel. Closing a channel can also help prevent deadlocks and resource leaks in programs that utilize goroutines for parallel processing. When a channel is closed, any ongoing receives will still complete but further sends will cause a panic. It is safe to close a channel only by the goroutine that created it. If you close a channel that has been closed or attempt to send on a closed channel, your program will crash.

# Example of closing a channel in Go

package main

import (
    "fmt"
    "time"
)

func main() {
    ch := make(chan string)

    go func() {
        for i := 0; i < 5; i++ {
            ch <- fmt.Sprintf("Message %d", i)
            time.Sleep(100 * time.Millisecond)
        }
        close(ch) // Closing the channel
    }()

    for msg := range ch { // Receiving messages until channel is closed
        fmt.Println(msg)
    }
}

# Example demonstrating panic on sending to a closed channel

package main

import "fmt"

func main() {
    ch := make(chan int)
    close(ch) // Close channel first
    // ch <- 1 // Uncommenting this line will cause panic
    fmt.Println("Channel closed")
}

# Example of a goroutine waiting on a closed channel

package main

import (
    "fmt"
    "time"
)

func main() {
    ch := make(chan int)

    go func() {
        for i := 0; i < 5; i++ {
            ch <- i
            time.Sleep(50 * time.Millisecond)
        }
        close(ch) // Close channel when done sending
    }()

    for msg := range ch { // Range will exit when channel is closed
        fmt.Println(msg)
    }
}
