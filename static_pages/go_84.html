cap

In Go, the `cap` function is used to return the capacity of a slice, which is the total number of elements the slice can hold without requiring a resize. The capacity of a slice is determined when it is initialized and can grow as needed, but it is always greater than or equal to its length. Understanding how capacity works is important for optimizing memory usage and performance in Go programs.

```python
# Example of using cap in Go


# Creating a slice in Go

# The slice 'mySlice' has an initial capacity of 5.
mySlice = make([]int, 0, 5)

# Printing the capacity of the slice using cap function
print(cap(mySlice))  # Output will be 5
```
```python
# Example of increasing the capacity of a slice


# Start with a slice of size 2 (length=2, capacity=2)
nums = []int{1, 2}

# Append additional elements to the slice
nums = append(nums, 3, 4)

# Checking capacity after appending
print(cap(nums))  # The output will depend on the underlying array's capacity ('may be greater than 2')
```
```python
# Example of creating a slice with a specific capacity


# Create a slice with capacity of 10
arr = make([]int, 0, 10)

# Check its initial capacity
print(cap(arr))  # Output will be 10

# Add 10 elements to fill it to its capacity
for i in range(10):
    arr = append(arr, i)

# Check capacity again after filling
print(cap(arr))  # Output will still be 10 (capacity does not change unless exceeded)
```
