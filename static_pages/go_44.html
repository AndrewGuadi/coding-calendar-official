cap

The 'cap' method in Go is used to determine the capacity of a slice. The capacity of a slice is defined as the number of elements the slice can hold before needing to allocate more memory. This method is particularly useful for optimizing memory usage and understanding how slices grow in Go. It is important to note that the capacity is always greater than or equal to the length of the slice, and can be checked using the built-in 'cap()' function.

s = []int{1, 2, 3}
length = len(s)  # Output: 3
capacity = cap(s)  # Output: 3

# Creating a larger slice.
s = make([]int, 0, 5)  # Length: 0, Capacity: 5
length = len(s)  # Output: 0
capacity = cap(s)  # Output: 5
s = []string{'a', 'b', 'c', 'd'}
length = len(s)  # Output: 4
capacity = cap(s)  # Output: 4

# Doubling the slice
s = append(s, 'e', 'f')  # Now length is 6
length = len(s)  # Output: 6
capacity = cap(s)  # Output: 8 (if reallocation happened)
s = make([]float64, 2, 10)  # Length: 2, Capacity: 10
length = len(s)  # Output: 2
capacity = cap(s)  # Output: 10

# Filling the slice
s[0] = 3.14
s[1] = 2.71

# Checking again
length = len(s)  # Output: 2
capacity = cap(s)  # Output: 10
