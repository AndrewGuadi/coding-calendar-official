close

The 'close' method in Go is commonly used to release resources such as files, network connections, or channels. When invoked on an object, it signals that no further operations will be performed with that object. For example, closing a file ensures that all buffered data is flushed and the file descriptor is released, making it available for other processes or activities. It's crucial to call 'close' to avoid resource leaks, especially in applications that handle numerous resources or when using limits (like file descriptors). When working with Go, 'close' is often invoked within deferred functions to ensure it runs when a surrounding function exits, regardless of the exit reason (normal return, panic, etc.).

import os

# Open a file and ensure it gets closed afterwards
file_path = 'sample.txt'

# Attempting to open the file
file = os.open(file_path, os.O_RDWR | os.O_CREAT)

try:
    # Perform file operations (read, write)
    os.write(file, b'Sample data')
finally:
    # Closing the file
    os.close(file)
import socket

# Create a socket for network communication
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

try:
    # Connect to an address
    s.connect(('localhost', 8080))
    # Send some data
    s.sendall(b'Hello, world')
finally:
    # Closing the socket
    s.close()
from queue import Queue

# Create a queue
q = Queue()

# Adding items to the queue
q.put('task1')
q.put('task2')

try:
    # Processing items from the queue
    while not q.empty():
        item = q.get()
        print(f'Processing {item}')
finally:
    # Closing resources - queues do not have a close method, but if this were a custom resource, we would implement cleanup here
    print('All tasks processed and resources cleaned up.')
