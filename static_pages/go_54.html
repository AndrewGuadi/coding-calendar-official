cap

In Go, the 'cap' function is used to return the capacity of a slice. Capacity is the total number of elements that the slice can hold before needing to allocate more memory. It is important for understanding the underlying array that a slice is referencing, and it can help optimize memory usage and performance when dealing with dynamic data structures.

def example_1():
    # Creating a slice (list) with initial values
    my_slice = [1, 2, 3, 4, 5]
    # Simulating the Go 'cap' function by determining capacity
    capacity = len(my_slice)  # Since Python lists and Capacity are synonymous here
    print(f'Capacity of my_slice: {capacity}')  # Output: Capacity of my_slice: 5

def example_2():
    # Creating an empty list
    my_empty_slice = []
    # Simulating the Go 'cap' for an empty slice
    capacity = len(my_empty_slice)
    print(f'Capacity of my_empty_slice: {capacity}')  # Output: Capacity of my_empty_slice: 0

def example_3():
    # Creating a list and extending it to demonstrate capacity changes
    my_list = [1, 2, 3]
    print('Initial list:', my_list)
    # Append items and print new length which indicates used size
    for i in range(4, 10):
        my_list.append(i)
        # Simulating capacity check after appending items
        capacity = len(my_list)  # Again treating this as capacity check
        print(f'After adding {i}, capacity of my_list: {capacity}')  # Dynamic capacity simulation
