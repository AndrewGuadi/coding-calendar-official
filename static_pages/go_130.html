cap

The 'cap' method in Go is used to return the capacity of a slice, which is the total number of elements that the slice can hold without requiring reallocation. Capacity may be equal to or greater than the length of the slice; when the length exceeds the capacity, a new underlying array will be allocated. This method is useful for understanding the memory allocation and sizing of slices when optimizing performance or when you need to manage memory efficiently.

```python
# Example 1: Understanding slice capacity in Go
# In Go, if you create a slice with a specific length and capacity
# you can check its capacity using the cap function.
# A demonstration would be:
slice := make([]int, 3, 5)  # Length is 3, capacity is 5
fmt.Println(cap(slice))  # Outputs: 5
```
```python
# Example 2: Changing the size of a slice
# When you append to a slice and its length exceeds the initial capacity,
# a new array is allocated and capacity increases. 
# Demonstration:
slice := []int{1, 2, 3}
fmt.Println(cap(slice))  # Outputs: 3
slice = append(slice, 4, 5)
fmt.Println(cap(slice))  # Might output: 6 (if a new array was allocated)
```
```python
# Example 3: Preallocating a slice for efficiency
# Using cap to optimize memory allocation by pre-defining a slice capacity.
# Demonstration:
prealloc := make([]int, 0, 10)  # Preallocate with capacity of 10
for i in range(10):
    prealloc = append(prealloc, i)
fmt.Println(cap(prealloc))  # Outputs: 10
```
