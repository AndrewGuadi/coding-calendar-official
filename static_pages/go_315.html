close

The 'close' method in Go is used to close a file or other resource that was previously opened. Closing a resource frees up any allocated resources associated with it. In the context of file handling, it is crucial to call 'close' to ensure that all buffered data is flushed and any locks on the file are released. Implementing proper error handling while closing resources is also essential since there may be cases where closing a file could fail due to various issues, such as underlying hardware problems or if the file descriptor is already closed.

import os

# Open a file in write mode
file = open('example.txt', 'w')

try:
    # Write some contents to the file
    file.write('Hello, World!')
finally:
    # Always close the file
    file.close()

# Check if the file is closed
print('File closed:', file.closed)
import json

# Create a JSON file and write data to it
with open('data.json', 'w') as json_file:
    json.dump({'key': 'value'}, json_file)
# The file is automatically closed after the with block

# Check if the file is closed
print('File closed:', json_file.closed)
import sqlite3

# Connect to a SQLite database
connection = sqlite3.connect('example.db')

try:
    # Perform database operations
    cursor = connection.cursor()
    cursor.execute('CREATE TABLE IF NOT EXISTS users (id INTEGER PRIMARY KEY, name TEXT)')
    cursor.execute('INSERT INTO users (name) VALUES (?)', ('Alice',))
    connection.commit()
finally:
    # Ensure the connection is closed
    connection.close()

# Check if the connection is closed
print('Connection closed:', connection.closed)
