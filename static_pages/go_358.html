close

In Go, the 'close' method is used to close a channel, which signals that no more values will be sent on it. It is a crucial method to prevent memory leaks and resource wastage, especially in concurrent programming scenarios where goroutines may still be trying to read from the channel after it has been closed. Once a channel is closed, any attempt to send values to it will cause a panic. However, it is safe to read from a closed channel; it will return the zero value of the channel's type and a boolean indicating that the channel is closed.

```go
package main

import (
	"fmt"
)

func main() {
	myChannel := make(chan string)
	
	// Goroutine to send messages
	go func() {
		myChannel <- "Hello"
		myChannel <- "World"
		close(myChannel) // Closing the channel
	}()

	// Receiving messages
	for msg := range myChannel {
		fmt.Println(msg)
	}
}
```
```go
package main

import (
	"fmt"
)

func main() {
	numbers := make(chan int)
	
	// Function to generate Fibonacci numbers
	go func() {
		a, b := 0, 1
		for i := 0; i < 10; i++ {
			numbers <- a
			a, b = b, a+b
		}
		close(numbers) // Closing the channel after sending all numbers
		
	}()
	
	// Receiving Fibonacci numbers
	for num := range numbers {
		fmt.Println(num)
	}
}
```
```go
package main

import (
	"fmt"
)

func main() {
	dataChannel := make(chan int)
	
	// Goroutine to send square of numbers
	go func() {
		for i := 1; i <= 5; i++ {
			dataChannel <- i * i
		}
		close(dataChannel) // Closing the channel
	}()
	
	// Receiving squared numbers
	for square := range dataChannel {
		fmt.Println(square)
	}
}
```
