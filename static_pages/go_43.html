cap

In Go, the 'cap' function is used to return the capacity of a slice, which is the number of elements the slice can hold without having to allocate more memory. The capacity is always greater than or equal to the length of the slice. This is a useful function for understanding the underlying storage characteristics of a slice, especially when making performance-related decisions or when managing memory in a Go program.

s := make([]int, 0, 10)
fmt.Println(cap(s)) // Outputs: 10

// In this example, we create a slice with a capacity of 10 but a length of 0. Using 'cap', we can check the capacity which is 10.
s := []string{"apple", "banana", "cherry"}

// The length of the slice is 3, but the capacity might be more, depending on the slice's initialization.
fmt.Println(cap(s)) // Outputs: 3 or more depending on the underlying array

// Here, we declare a slice of strings and check its capacity using 'cap'.
s := []float64{1.1, 2.2, 3.3}

// Printing capacity before appending
fmt.Println(cap(s)) // Outputs: 3 or 4 depending on the Go implementation

s = append(s, 4.4, 5.5)

// Now we check the capacity after appending elements
fmt.Println(cap(s)) // It could increase, showing the new capacity of the slice if memory allocation happened.
