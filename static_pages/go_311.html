close

In Go, the 'close' method is used to close a channel, which prevents further sends on that channel. Once a channel is closed, it cannot be reopened, and any operations attempting to send on a closed channel will panic. However, receiving from a closed channel can still be done, and it will return a zero value along with a boolean 'false' to indicate that the channel is closed. Closing a channel is essential to signal to the receiving goroutines that no more values will be sent. It is important to note that only the sender should close a channel, not the receiver.

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class CloseChannelExample {
    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(2);
        // Execute tasks in separate threads
        executor.submit(() -> {
            // Simulate work
            for (int i = 0; i < 5; i++) {
                System.out.println("Sending: " + i);
            }
            // Close channel
            System.out.println("Channel closed.");
        });
        executor.shutdown();
    }
}
