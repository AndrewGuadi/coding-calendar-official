close

In Go, the 'close' method is used to close channels. It is an important part of concurrent programming in Go, as closing a channel signals that no more values will be sent on it. This is particularly useful for range loops or blocking receives, where you need to know when to stop listening for values. Once a channel is closed, any attempt to send values to it will cause a panic. However, it is safe to read from a closed channel; it will return the zero value of the channel's type until the channel is garbage collected.

import threading
import time

def worker(ch):
    print('Worker started and waiting for values...')

```python
import threading
import time

def producer(ch, n):
    for i in range(n):
        print(f'Producing: {i}')
        ch.put(i)
        time.sleep(1)
    ch.put(None)  # Signal that production is done

ch = queue.Queue()

# Create a producer thread
producer_thread = threading.Thread(target=producer, args=(ch, 5))
producer_thread.start()

# Consume the produced items
while True:
    item = ch.get()
    if item is None:
        break  # Stop consuming when we've hit the signal
    print(f'Consumed: {item}')

producer_thread.join()  # Ensure the producer thread has finished
```
```python
import threading
import time

def event_handler(event):
    print('Event handler waiting...')
    event.wait()  # Wait for the event to be set
    print('Event received, processing now!')

# Create an event
event = threading.Event()

# Start event handler thread
handler_thread = threading.Thread(target=event_handler, args=(event,))
handler_thread.start()

# Main thread sleeping for a bit
time.sleep(3)
print('Main thread setting the event...')
# Signal event
event.set()

handler_thread.join()  # Wait for the handler to finish
```
