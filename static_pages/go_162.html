cap

In Go, the 'cap' function is used to obtain the capacity of a slice. The capacity is the number of elements in the underlying array that the slice can hold. It gives insight into the performance and memory usage of slice operations. When you append elements to a slice, if the number of elements exceeds its capacity, Go automatically allocates a new larger array and copies the existing elements over, which can incur performance costs. Understanding the capacity can help optimize memory allocation and performance.

my_slice = [1, 2, 3]
# Get slice capacity
cap(my_slice) # Output: 3

# Appending more elements
my_slice.append(4)
print(my_slice) # Output: [1, 2, 3, 4]

# Return new capacity after append
cap(my_slice) # Output may vary based on system and Go implementation.
my_slice = ['apple', 'banana', 'cherry']
# Get initial capacity
print('Initial Capacity:', cap(my_slice)) # Output: Initial Capacity: 3

# Increase capacity by appending more elements
my_slice += ['date', 'elderberry']
print('New Slice:', my_slice) # Output: New Slice: ['apple', 'banana', 'cherry', 'date', 'elderberry']
print('New Capacity:', cap(my_slice)) # Output may vary
my_slice = [10, 20]
# Capacity before append
print('Old Capacity:', cap(my_slice)) # Output: Old Capacity: 2

# Appending new values
my_slice.append(30)
print('Slice after append:', my_slice) # Output: Slice after append: [10, 20, 30]
print('New Capacity:', cap(my_slice)) # Output may vary
